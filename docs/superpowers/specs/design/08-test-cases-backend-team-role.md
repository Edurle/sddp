# 后端接口测试用例 — 团队、成员、邀请、角色

> 用例编号格式：`TC-TEAM-{序号}`、`TC-MEMBER-{序号}`、`TC-INVITE-{序号}`、`TC-ROLE-{序号}`

---

## 1. 团队管理

### 创建团队

**接口**: `POST /api/v1/teams`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-TEAM-001 | 创建团队成功 | 已登录 | `{ "name": "团队A", "description": "描述" }` | 0 | `{ "id": 1, "name": "团队A" }` |
| TC-TEAM-002 | 创建者自动成为所有者 | 已登录 | 同上 | 0 | 创建者为 owner |
| TC-TEAM-003 | 名称为空 | 已登录 | `{ "name": "", "description": "" }` | 40001 | 参数校验失败 |
| TC-TEAM-004 | 未登录 | 未登录 | `{ "name": "团队A" }` | 40100 | 未登录 |

### 获取团队详情

**接口**: `GET /api/v1/teams/{id}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-TEAM-005 | 获取成功 | 团队成员 | 路径 id=1 | 0 | 返回团队详情含 owner 和 member_count |
| TC-TEAM-006 | 团队不存在 | 已登录 | 路径 id=999 | 40400 | 资源不存在 |
| TC-TEAM-007 | 非成员无权查看 | 已登录但非成员 | 路径 id=1 | 40300 | 无权限 |

### 编辑团队

**接口**: `PUT /api/v1/teams/{id}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-TEAM-008 | 所有者编辑成功 | 团队所有者 | `{ "name": "新名称", "description": "新描述" }` | 0 | 更新成功 |
| TC-TEAM-009 | 管理员编辑成功 | 团队管理员 | 同上 | 0 | 更新成功 |
| TC-TEAM-010 | 普通成员无权编辑 | 普通成员 | 同上 | 40300 | 无权限 |
| TC-TEAM-011 | 名称为空 | 团队所有者 | `{ "name": "" }` | 40001 | 参数校验失败 |

### 解散团队

**接口**: `DELETE /api/v1/teams/{id}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-TEAM-012 | 解散成功（软删除） | 团队所有者 | Header: X-Confirm-Delete=团队名 | 0 | 团队已解散 |
| TC-TEAM-013 | 确认信息不匹配 | 团队所有者 | Header: X-Confirm-Delete=错误名 | 40001 | 确认信息不匹配 |
| TC-TEAM-014 | 非所有者拒绝 | 管理员/成员 | Header: X-Confirm-Delete=团队名 | 40300 | 无权限 |
| TC-TEAM-015 | 解散后查询返回不存在 | 团队已解散 | GET /teams/{id} | 40400 | 资源不存在 |

### 转让团队

**接口**: `POST /api/v1/teams/{id}/transfer`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-TEAM-016 | 转让成功 | 团队所有者 | `{ "new_owner_id": 2 }` | 0 | 团队已转让 |
| TC-TEAM-017 | 新所有者不在团队中 | 团队所有者 | `{ "new_owner_id": 999 }` | 40400 | 用户不在团队中 |
| TC-TEAM-018 | 非所有者拒绝 | 管理员 | `{ "new_owner_id": 2 }` | 40300 | 无权限 |
| TC-TEAM-019 | 转让给自己 | 团队所有者 | `{ "new_owner_id": 自己id }` | 40001 | 不能转让给自己 |

## 2. 团队成员管理

### 获取成员列表

**接口**: `GET /api/v1/teams/{id}/members`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-MEMBER-001 | 获取成功 | 团队成员 | 无 | 0 | 返回成员列表含角色信息 |
| TC-MEMBER-002 | 按角色筛选 | 团队成员 | ?role_id=1 | 0 | 仅返回该角色成员 |
| TC-MEMBER-003 | 已软删除的成员不显示 | 有已移出成员 | 无 | 0 | 列表不含已移出成员 |

### 邀请成员

**接口**: `POST /api/v1/teams/{id}/invitations`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-MEMBER-004 | 邀请成功 | 有 member:invite 权限 | `{ "identifier": "user1" }` | 0 | 邀请已发送 |
| TC-MEMBER-005 | 用户不存在 | 同上 | `{ "identifier": "不存在" }` | 40400 | 用户不存在 |
| TC-MEMBER-006 | 用户已在团队中 | 同上 | `{ "identifier": "已有成员" }` | 40006 | 用户已在团队中 |
| TC-MEMBER-007 | 已有待处理邀请 | 同上 | `{ "identifier": "已邀请用户" }` | 40001 | 已有待处理邀请 |
| TC-MEMBER-008 | 无权限邀请 | 普通成员无权限 | `{ "identifier": "user1" }` | 40300 | 无权限 |

### 移出成员

**接口**: `DELETE /api/v1/teams/{teamId}/members/{userId}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-MEMBER-009 | 移出成功（软删除） | 有 member:remove 权限 | 路径 userId | 0 | 成员已移出 |
| TC-MEMBER-010 | 不能移出所有者 | 同上 | 路径 userId=owner | 40008 | 不能移出团队所有者 |
| TC-MEMBER-011 | 无权限移出 | 普通成员 | 路径 userId | 40300 | 无权限 |
| TC-MEMBER-012 | 移出后不在成员列表 | 成员已移出 | GET /members | 0 | 列表不含该成员 |

### 分配角色

**接口**: `PUT /api/v1/teams/{teamId}/members/{userId}/roles`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-MEMBER-013 | 分配角色成功 | 有 member:assign_role 权限 | `{ "role_ids": [1, 2] }` | 0 | 角色已更新 |
| TC-MEMBER-014 | 角色不存在 | 同上 | `{ "role_ids": [999] }` | 40400 | 角色不存在 |
| TC-MEMBER-015 | 多角色权限取并集 | 分配多个角色 | `{ "role_ids": [1, 2] }` | 0 | 用户拥有两角色全部权限 |

## 3. 邀请管理

### 获取我的邀请

**接口**: `GET /api/v1/invitations/pending`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-INVITE-001 | 获取成功 | 有待处理邀请 | 无 | 0 | 返回邀请列表 |
| TC-INVITE-002 | 无邀请 | 无邀请 | 无 | 0 | 返回空列表 |

### 处理邀请

**接口**: `PUT /api/v1/invitations/{id}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-INVITE-003 | 接受邀请 | 有 pending 邀请 | `{ "action": "accept" }` | 0 | 加入团队成功 |
| TC-INVITE-004 | 拒绝邀请 | 有 pending 邀请 | `{ "action": "reject" }` | 0 | 邀请已拒绝 |
| TC-INVITE-005 | 邀请不存在 | 已登录 | 路径 id=999 | 40400 | 邀请不存在 |
| TC-INVITE-006 | 已处理过的邀请 | 邀请 status=accepted | `{ "action": "accept" }` | 40005 | 邀请已处理 |
| TC-INVITE-007 | 不是被邀请人 | 登录用户非 invitee | `{ "action": "accept" }` | 40300 | 无权操作 |

## 4. 角色管理

### 获取角色列表

**接口**: `GET /api/v1/teams/{teamId}/roles`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-ROLE-001 | 获取成功 | 团队成员 | 无 | 0 | 返回角色列表含权限项 |
| TC-ROLE-002 | 包含内置角色 | 新团队 | 无 | 0 | 列表含团队所有者和团队管理员 |

### 创建角色

**接口**: `POST /api/v1/teams/{teamId}/roles`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-ROLE-003 | 创建成功 | 有 member:assign_role 权限 | `{ "name": "开发者", "description": "...", "permissions": ["task:create", "task:edit"] }` | 0 | `{ "id": 3 }` |
| TC-ROLE-004 | 名称重复 | 同团队已有同名角色 | `{ "name": "开发者", ... }` | 40007 | 角色名称已存在 |
| TC-ROLE-005 | 名称为空 | 同上 | `{ "name": "", ... }` | 40001 | 参数校验失败 |
| TC-ROLE-006 | 无权限 | 普通成员 | 同 TC-ROLE-003 | 40300 | 无权限 |

### 编辑角色

**接口**: `PUT /api/v1/teams/{teamId}/roles/{roleId}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-ROLE-007 | 编辑成功 | 有权限，非内置角色 | `{ "name": "高级开发者", "permissions": [...] }` | 0 | 更新成功 |
| TC-ROLE-008 | 内置角色不可修改 | 编辑内置角色 | 同上 | 40009 | 内置角色不可修改 |
| TC-ROLE-009 | 角色不存在 | roleId=999 | 同上 | 40400 | 角色不存在 |
| TC-ROLE-010 | 已软删除的角色不可修改 | 角色已删除 | 同上 | 40400 | 角色不存在 |

### 删除角色

**接口**: `DELETE /api/v1/teams/{teamId}/roles/{roleId}`

| 用例编号 | 描述 | 前置条件 | 请求参数 | 预期响应 code | 预期响应 data/说明 |
|----------|------|----------|----------|---------------|-------------------|
| TC-ROLE-011 | 删除成功（软删除） | 有权限，非内置角色 | 路径 roleId | 0 | 角色已删除 |
| TC-ROLE-012 | 内置角色不可删除 | 删除内置角色 | 路径 roleId=1 | 40009 | 内置角色不可删除 |
| TC-ROLE-013 | 删除后不在角色列表 | 角色已删除 | GET /roles | 0 | 列表不含已删除角色 |
