# 需求详情页重新设计

## 背景

当前 `RequirementDetailPage.vue` 存在以下问题：
- 单文件 786 行，所有功能堆在一起
- 单列堆叠布局，宽屏空间浪费
- 操作按钮散落在各处
- Tab 内容（规范编辑、测试用例表格）缺乏视觉层次
- 步骤导航为横向胶囊，占用水平空间

## 设计

### 整体布局：双栏

```
┌─────────────────────────────────────────────────┐
│ ┌──────────┐ ┌──────────────────────────────────┐│
│ │ 左侧面板  │ │ 右侧内容区                        ││
│ │ (280px)  │ │                                  ││
│ │          │ │ ┌─ Tab 导航 ─────────────────┐   ││
│ │ 进度步骤  │ │ │ 规范 │ 版本(3) │ 任务 │ 测试 │   ││
│ │          │ │ └────────────────────────────┘   ││
│ │ 基本信息  │ │                                  ││
│ │          │ │ ┌─ Tab 内容 ─────────────────┐   ││
│ │ 审核历史  │ │ │                              │   ││
│ │          │ │ │  （规范编辑器 / 表格 / 卡片）│   ││
│ │ 操作按钮  │ │ │                              │   ││
│ │          │ │ └────────────────────────────┘   ││
│ │          │ │                                  ││
│ │          │ │ ┌─ 统计卡片 ─────────────────┐   ││
│ │          │ │ │ 5用例 │ 4通过 │ 1失败 │ 80% │   ││
│ │          │ │ └────────────────────────────┘   ││
│ └──────────┘ └──────────────────────────────────┘│
└─────────────────────────────────────────────────┘
```

### 左侧面板

固定宽度 280px，包含四个区块，纵向排列：

1. **进度步骤**（竖向）
   - 4 个步骤：需求 → 规范 → 测试 → 已通过
   - 当前步骤蓝色高亮，已完成步骤深色+勾号，未到达步骤灰色
   - 每步显示状态文字标签

2. **基本信息卡片**
   - 标题（大字）
   - 类型 Badge + 优先级 Badge（横排）
   - 描述文字
   - Bug 类型显示复现步骤/环境/严重度
   - 优化类型显示当前问题/预期改善
   - 驳回时显示驳回原因（红色背景）

3. **审核历史**
   - 紧凑时间线样式
   - 每条显示：操作（通过/驳回）+ 审核人 + 时间
   - 驳回时附带原因

4. **操作按钮**
   - 竖向排列，占满宽度
   - 根据需求状态动态显示：编辑、保存、删除、提交审核、通过、驳回

### 右侧内容区

1. **Tab 导航** — 下划线式（非按钮式）
   - 规范、版本历史(N)、任务(N)、测试用例(N)
   - 规范 tab 始终可见；其他 tab 显示计数
   - 根据状态自动选中：drafting_spec/reviewing_spec → 规范；drafting_tests/reviewing_tests → 测试用例；approved → 任务

2. **规范 Tab**
   - 顶部工具栏：版本信息 + 保存按钮 + 提交审核按钮
   - 内容区 textarea（保留，但样式改进：等宽字体、更大行高、内边距）
   - 保存成功提示在工具栏内联显示

3. **版本历史 Tab**
   - 列表卡片形式，每个版本一张卡片
   - 卡片显示：版本号、时间、内容预览
   - 点击卡片展开查看完整内容
   - 替换当前的按钮列表+纯文本展示

4. **任务 Tab**
   - 保留表格，但操作按钮改为图标/链接风格
   - 顶部添加任务按钮

5. **测试用例 Tab**
   - 顶部：添加按钮 + 类型筛选下拉 + 提交审核按钮
   - 表格保留，操作列按钮改为文字链接风格
   - 表格行增加序号色块标识

6. **统计卡片**（固定在内容区底部）
   - 4 个数字卡片横排：总用例数、通过数、失败数、通过率
   - 移除单独的"测试统计" tab，合并为底部常驻展示

### 组件拆分

从单文件拆为：

| 文件 | 职责 |
|------|------|
| `RequirementDetailPage.vue` | 主布局（双栏框架）+ 数据获取 + 状态管理 |
| `RequirementSidebar.vue` | 左侧面板：进度、信息、审核历史、操作 |
| `SpecEditor.vue` | 规范编辑 tab |
| `SpecVersions.vue` | 版本历史 tab |
| `TaskList.vue` | 任务 tab |
| `TestCaseList.vue` | 测试用例 tab |
| `TestStatCards.vue` | 统计卡片 |

所有弹窗保留在 `RequirementDetailPage.vue` 中（或跟随对应 tab 组件）。

### data-testid 兼容性

保留所有现有 testid 不变，确保 E2E 测试（E2E-REQ-001 ~ E2E-REQ-016）全部兼容。移除的"测试统计" tab（`req-detail-tab-test-stats`）改为统计卡片，testdata 保留但位置从 tab 内容变为底部常驻区域。

### 窄屏适配

当屏幕宽度 < 768px 时，左侧面板折叠为顶部区域，双栏变单栏。
